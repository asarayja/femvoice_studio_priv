- ComfortZoneController (dynamic pitch safety)

**Primary Objective**

Transform the exercise system into clinically intelligent, real-time adaptive biofeedback modules that dynamically respond to microphone input and integrate SmartCoach inline within exercise UI (no separate SmartCoach window).

**Clinical Logic Rules with Adaptive Thresholds**

Implement exercise-specific evaluation logic using dynamically adapted thresholds:

**Resonance Exercises:**
- Primary metric = ResonanceScore (evaluated against user-specific formant targets based on individual vocal tract characteristics and spectral brightness)
- Secondary = Stability (threshold adapted per user based on control level and resonance consistency)
- Pitch validated only if outside ComfortZone

**Pitch Exercises:**
- Primary metric = PitchScore (validated against user-adapted comfort ranges, not fixed Hz)
- Secondary = Stability
- Resonance monitored but not blocking

**Intonation Exercises:**
- Primary = PitchSlope tracking
- Secondary = Range variability (adapted to user's natural vocal range)

**Straw Phonation:**
- Primary = Intensity + Stability
- Resonance lightly weighted

**Adaptive Threshold System:**
The coordinator must integrate with FemVoiceScoreEngine's user-specific adaptation logic:
- Resonance ranges dynamically adapted based on formant structure and spectral brightness
- Stability thresholds adjusted based on user's demonstrated control level and vocal health metrics
- All thresholds evolve with user progress rather than using fixed clinical targets

**Safety Integration Rules**

When `HealthScore < 70` OR `ComfortZoneController.IsSafetyLocked == true`:
- Disable score accumulation
- Display soft warning via InlineCoachMessage (Severity: Warning)
- Freeze hold timer (HoldProgress stops incrementing)
- Continue displaying real-time feedback but prevent progression
