## Service 1: FemVoiceScoreEngine.cs

**Purpose**: Calculate a dynamic FemVoiceScore that combines multiple metrics with adaptive normalization based on individual user progress patterns.

**Score Composition**

Base formula with configurable weights (define as constants with default values, allow config override):

```
FemVoiceScore = 
  0.45 × ResonanceScore +
  0.30 × PitchScore +
  0.15 × StabilityScore +
  0.10 × HealthModifier
```

**Adaptive Intelligence Requirements**

The engine must adapt scoring to individual user baselines:

- Track a 30-day rolling baseline per user
- Apply exponential smoothing for long-term trend calculation
- Normalize current performance against personal trend (not absolute scale)
- Detect stagnation: plateau lasting >14 days
- Detect regression: sustained >10% drop from baseline
- Prevent score inflation during unstable sessions

**Event Interface**

Expose:
```csharp
public event Action<FemVoiceScoreSnapshot> ScoreUpdated;
```

**FemVoiceScoreSnapshot Model** must include:
- `RawScore` (weighted composite)
- `AdaptiveScore` (normalized against baseline)
- `TrendSlope` (direction and rate of change)
- `PlateauDetected` (bool)
- `RegressionDetected` (bool)
- `Timestamp`

**Dependency Injection**

Constructor must accept:
- `IScoreRepository` for persisting baseline data
- `IUserRepository` for user-specific settings
- Any configuration object for weight overrides

**Technical Constraints**

- No static state
- No direct database access (use injected repository interfaces)
- No UI references
- All magic numbers replaced with named constants
- Full XML documentation
- Thread-safe event raising

## Service 2: ComfortZoneController.cs

**Purpose**: Dynamically manage safe pitch training boundaries that adapt to user performance and health status.

**Zone Management Rules**

The controller determines safe pitch ranges based on:
- ResonanceScore stability
- AdaptiveFemVoiceScore trends
- HealthScore thresholds
- PitchStability metrics

**Expansion Logic**

- Expand zone gradually when resonance stable ≥7 consecutive days
- Maximum expansion rate: 5% per week
- Expansion only allowed when all safety conditions met

**Contraction/Lock Logic**

- Immediate contraction on vocal strain detection or instability
- Freeze all progression if HealthScore <70
- Safety lock for 3 days after strain incident
- Maintain individualized zone width based on user tolerance data

**Event Interface**

Expose:
```csharp
public event Action<ComfortZoneState> ZoneUpdated;
```

**ComfortZoneState Model** must include:
- `MinPitch` (Hz)
- `MaxPitch` (Hz)
- `ZoneWidth` (Hz)
- `IsExpansionAllowed` (bool)
- `IsSafetyLocked` (bool)
- `Reason` (string explaining current state)

**Safety Decision Matrix**

| Condition | Action |
|-----------|--------|
| HealthScore <70 | Contract zone |
| Stability below threshold | Freeze expansion |
| Rapid score increase | Limited expansion only |
| Strain incident detected | 3-day safety lock |

**Dependency Injection**

Constructor must accept specific repository interfaces for zone persistence and user settings retrieval.
