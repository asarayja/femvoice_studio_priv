# Utvid FemVoice Studio med klinisk basert kompleksitetsnivå-system

Du er en senior C#/WPF-arkitekt med ekspertise innen sanntids lydanalyse, klinisk voice feminization therapy, adaptiv progresjonslogikk og MVVM-arkitektur. Du kjenner etablerte FemVoice Studio-mønstre fra eksisterende kode.

## Klinisk grunnlag

Voice feminization-trening krever integrert progresjon fra isolerte tekniske øvelser til naturlig, funksjonell tale. Pitch (F0), resonans, intonasjon, artikulasjon og spontan kommunikasjonsbruk må trenes sammen. Dette systemet skal modellere denne progresjonen klinisk korrekt.

## Din oppgave

Implementer et komplett `ComplexityEngine`-system som utvider eksisterende `ProgressionEngine` og `WeeklyPlannerEngine` med talebrukskompleksitet. Dette skal gi produksjonsklar C#-kode som følger FemVoice Studios etablerte mønstre.

## 1. ComplexityEngine-komponenten

### Enums og datastrukturer

```csharp
public enum SpeechComplexityLevel
{
    IsolatedSounds,      // Isolerte lyder (humming, "aaaa")
    Syllables,           // Stavelser ("la-la-la")
    Words,               // Enkeltord ("hei", "takk")
    Phrases,             // Korte fraser ("god morgen")
    StructuredSentences, // Fullendte setninger
    SpontaneousSpeech,   // Spontan tale med pauser
    Conversational       // Naturlig dialog
}

public class ComplexityEvaluation
{
    public SpeechComplexityLevel CurrentLevel { get; set; }
    public double SuccessRate { get; set; }        // 0-100, siste 5 økter på dette nivået
    public bool IsReadyForNext { get; set; }       // Oppfyller alle kriterier
    public string Feedback { get; set; }           // Pedagogisk forklaring
    public int SessionsAtCurrentLevel { get; set; }
    public DateTime LastLevelChange { get; set; }
}
```

### Progresjonslogikk (klinisk informert)

`ComplexityEngine` skal kun tillate nivåopprykk når **ALLE** disse kriteriene er oppfylt:

**Teknisk stabilitet:**
- Resonans score ≥60 (gjennomsnitt siste 3 økter)
- Pitch stability ≥60 (fra `LiveMetricsService.StabilityState`)
- Intonasjon score ≥50 (ikke flat tale)

**Helse og sikkerhet:**
- `VoiceHealthScore` ≥70
- Ingen `HealthStatus.Warning` eller `Critical` i siste 3 økter
- `StrainLevel` <50 (gjennomsnitt)

**Repeterbar suksess:**
- Minimum 5 økter fullført på nåværende nivå
- `SuccessRate` ≥70% (beregnet fra `SessionEvaluationSummary.OverallScore`)
- Minimum 2 økter per uke siste 2 uker (konsistens)

**Eksempel på metoder:**
```csharp
public ComplexityEvaluation EvaluateCurrentLevel(int userId)
public bool CanAdvanceToNextLevel(ComplexityEvaluation evaluation)
public SpeechComplexityLevel DetermineNextLevel(SpeechComplexityLevel current)
```

## 2. WeeklyPlannerEngine-utvidelse

`WeeklyPlannerEngine` skal ta `SpeechComplexityLevel` som input og:

### Øktfordeling basert på kompleksitet

**Eksempel på ukeplan:**

Hvis `CurrentLevel = SpeechComplexityLevel.Words`:

- **Mandag**: Words-øvelse (funksjonell tale, enkeltord-repetisjoner)
- **Tirsdag**: IsolatedSounds + Syllables (teknisk pitch/resonans-trening)
- **Onsdag**: Words → Phrases (progresjon mot neste nivå)
- **Fredag**: Words (konsolidering)

### Filtering og readiness-sjekk

Før ukeplanen genereres:
- Sjekk `ComplexityEvaluation.IsReadyForNext`
- Hvis `false`: planlegg kun økter på `CurrentLevel` eller lavere
- Hvis `true`: inkluder 1-2 økter på `NextLevel` som "preview"

### Øvelseskategorisering

Bruk `ExerciseTextService` og kategoriser øvelser per kompleksitetsnivå basert på `ExerciseText.Content`:

- **IsolatedSounds/Syllables**: Øvelser med ID 1-15 (Grunnleggende humming, Pitch exploration)
- **Words/Phrases**: Øvelser med ID 16-35 (enkle setninger, daglige fraser)
- **StructuredSentences+**: Øvelser med ID 36-50 (narrativ, emosjonell prosodi)

## 6. Arkitektur- og implementasjonskrav

### Følg etablerte mønstre

**Dependency Injection:**
- Constructor injection av `DatabaseService`, `LiveMetricsService`, `SmartCoachEngine`
- Null-sjekk med `?? throw new ArgumentNullException()`

**MVVM:**
- `[ObservableProperty]` attributes (MVVM Community Toolkit)
- `RelayCommand` for UI-actions
- ViewModels arver fra `ViewModelBase` eller `ObservableObject`

**Service-patterns:**
- Singleton pattern for `ComplexityEngine` (hvis stateless), alternativt dependency injection
- Event-driven: `ComplexityLevelChanged` event når nivå endres

**Feilhåndtering:**
- Try-catch med `Debug.WriteLine()` for logging
- Graceful fallback (returnér `CurrentLevel` hvis evaluering feiler)

### Dataintegritet

- All kompleksitetsdata persisteres via `DatabaseService`
- Transaksjonssikkerhet ved nivåendringer (UPDATE både ComplexityProgress og UserProgress)
- Migrasjonssikker skjemadesign (`AddColumnIfNotExists`-mønster)

### Ytelse

- Kompleksitetsevaluering skal kjøre asynkront (`async/await` pattern)
- Caching av siste evaluering i-memory (oppdateres etter hver økt)
- Unngå heavy queries i real-time feedback loops

## Viktige referanser fra eksisterende kode

**AdaptiveComfortZoneService.cs**: Følg progresjonsmønster (minimum 3 stabile økter før advancement), resonans-prioritering (≥60 threshold).

**DirectionAnalyzer.cs**: Bruk `DirectionAnalysisResult`-struktur for å gi parameter-spesifikk feedback (Resonance, Pitch, Intonation, VoiceHealth).

**ExerciseTextService.cs**: Kompleksitetsnivåer mapper til `Difficulty` + `TargetPitchVariation` + `TargetIntonationRise`. IsolatedSounds = lav variation (14-20), Conversational = høy (30-50).

**FemVoiceScore.cs**: Bruk samme scoring weights (Resonance 45%, Pitch 30%, Intonation 15%, Health 10%) i kompleksitetsevaluering.

**ProgressionService.cs**: Repliker safety lock pattern—blokkér kompleksitetsopprykk hvis `StrainLevel` ≥70 eller 2+ moderate strains (≥40%).

**SmartCoachSubsystem.cs**: Følg clinical priority hierarchy (Resonance > Health > Pitch > Intonation), generer `CoachMessage` med What/Why/How-struktur.
