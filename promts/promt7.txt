Du er en senior C#/WPF-arkitekt med spesialisering i sanntids lydanalyse, biofeedback-systemer og voice feminization training. Du har dyp kunnskap om evidensbaserte stemmetreningsprinsipper fra moderne logopedi og kjønn-affirmativ stemmeterapi.

**Din oppgave:** Implementer et komplett Smart Coach- og sanntids feedback-system for FemVoice Studio som følger kliniske voice feminization-prinsipper, integrerer med eksisterende arkitektur, og prioriterer stemmehelse.

## Plattformkontekst

FemVoice Studio er en adaptiv stemmetreningsplattform i C#/WPF som støtter pitch, resonans, intonasjon og progresjon. Eksisterende arkitektur følger MVVM-mønster med NAudio for lydopptak (44.1kHz, mono, 16-bit), OxyPlot for sanntidsvisualisering, og LocalizationService for flerspråklig støtte (norsk/engelsk).

**Eksisterende komponenter du skal integrere med:**
- `AudioCaptureService` – lydopptak med callback-pattern
- `FormantDetectionService` – 25ms window, 10ms hop, formant-ekstraksjon
- `DatabaseService` – SQLite persistence (MyDocuments/FemVoiceStudio/femvoice.db)
- `ProgressionSubsystem` – historisk data (ProgressionDashboard.xaml.cs, ProgressionService.cs)
- `ExerciseFeedbackEngine` – evaluering mot øvelsesdefinisjoner (ExerciseFeedbackEngine.cs)
- `SmartCoachEngine` – baseline-beregning, progresjonsevaluering (SmartCoachEngine.cs)
- `DirectionAnalyzer` – klinisk beslutningsmottor for parameterretning (DirectionAnalyzer.cs)
- `FemVoiceScore` – vektet scoring (resonans 45%, pitch 30%, intonasjon 15%, stemmehelse 10%)

Kodebasen bruker CommunityToolkit.Mvvm (ObservableObject, RelayCommand) og DispatcherTimer for UI-oppdateringer.

## Kliniske prinsipper (kritisk)

**Resonans-først-hierarki:** Resonans prioriteres ALLTID over pitch. Hvis F2 < 1400 Hz (bakre resonans), må resonansforbedring komme før pitch-justering. Dette gjenspeiler evidensbasert voice feminization: forward resonance er fundamentet for feminin stemme-oppfatning.

**Stemmehelse som gatekeeper:** Strain-deteksjon (jitter, shimmer, amplitude spikes, pitch press >180 Hz) blokkerer progresjon. Kritisk strain (>75) skal automatisk pause øvelser og vise hvile-instruksjoner.

**Adaptive målsoner:** Ikke statiske terskler. Baseline beregnes fra minimum 3 økter (ideelt 14+ dager). Målsoner justeres gradvis basert på brukerens progresjon—maksimalt 5 Hz pitch-økning per økt, 2 Hz/uke default progresjon.

**Naturlighet over ekstremitet:** Målsettingen er en stemme brukeren kan opprettholde uten strain, ikke maksimal pitch. Vektlegg konsistens og forward resonance fremfor absolutte Hz-verdier.

## Implementasjonskrav

### 1. Sanntids analysemotor

Utvid eksisterende lydanalyse-pipeline med holistisk evaluering:

**Parametere å spore (alle nødvendige for klinisk analyse):**
- **Pitch (F0):** Målområde 165–255 Hz (begynner 150–200, middels 160–240, avansert 165–255). Spor min/max/avg per økt. Detekter pitch press (>180 Hz + ustabilitet).
- **Resonans (formanter):** F1 400–700 Hz (optimal 500), F2 1400–2200 Hz (optimal 1800), F3 2500–3500 Hz. Beregn forward/neutral/back kategorisering (ResonansScoringService-logikk). Spor spectral centroid (>2000 Hz = bright tone).
- **Intonasjon/Prosodi:** Pitch range (Hz variasjon), intonation rise score (0.0–1.0), ideal range 30–120 Hz. Detekter flat prosodi (<30 Hz = monoton).
- **Stabilitet:** Jitter (freq variasjon), shimmer (amplitude variasjon), HNR (harmonics-to-noise ratio). Terskler: jitter >1.5%, shimmer >3.5%, HNR <15 dB indikerer strain.
- **Pust/Flow:** Intensity RMS (0.0–1.0), luftflow-indikatorer. Penaliser >0.8 (for kraftig) eller <0.01 (for svak).

**Latenskrav:** Evaluering og UI-oppdatering <100 ms for effektiv biofeedback. Bruk 50ms throttling (se LiveFeedbackViewModel.cs) for å balansere responsivitet og ytelse.

**Output:** Produser `VoiceMetrics`-objekt per analysesyklus med alle parametere + timestamp + isInRange-boolean per parameter.

### 2. Feedback-regelmotor

Implementer regelbasert evaluering per øvelsestype som returnerer `EvaluationStatus` (Correct/Adjust/Stop) + localized coach hint key.

**Pitch-øvelser (kategori: "pitch"):**
- Evaluer F0 mot øvelsens `TargetMinPitch`/`TargetMaxPitch` ± effektiv toleranse (begynner ±10 Hz, middels ±7, avansert ±5)
- Detekter press: pitch >180 Hz + jitter >1.5% → Stop + "CoachHint_PitchPress_Rest"
- Detekter ustabilitet: pitch variasjon >25 Hz → Adjust + "CoachHint_Pitch_Stabilize"
- Korrekt: F0 innenfor target ± toleranse + stabilitet OK → Correct + "CoachHint_Pitch_Good"

**Resonansøvelser (kategori: "resonans"):**
- Prioriter F2-evaluering: F2 < 1400 Hz → Adjust + "CoachHint_Resonance_MoveForward"
- Optimal: F2 1800–2200 Hz + F1 400–700 Hz → Correct + "CoachHint_Resonance_Optimal"
- Intermediær: F2 1400–1800 Hz → Adjust + "CoachHint_Resonance_AlmostThere"
- Evaluer spectral centroid: <2000 Hz → hint om lysere tone

**Intonasjonsøvelser (kategori: "intonasjon"):**
- Krav: pitch range >30 Hz (ikke monoton)
- Spørsmålsøvelser: sjekk rising intonation (intonation rise score >0.2)
- Utsagnsøvelser: sjekk falling intonation (negativ slope i siste 30% av frase)
- Flat: range <30 Hz → Adjust + "CoachHint_Intonation_AddVariation"
- Natural: range 40–120 Hz + passende slope → Correct + "CoachHint_Intonation_Natural"

**Pustøvelser (kategori: "pust"):**
- Evaluer intensity RMS jevnhet (lav variance = god pustekontroll)
- Sjekk frasevarighetspredikabilitiet
- Detekter luftmangel: intensity RMS synker >30% mot slutten → Adjust + "CoachHint_Breathing_DeepBreath"

**Tverrgående strain-deteksjon (prioritet over alt):**
- Jitter >2.5% ELLER shimmer >5.0% ELLER amplitude spike >2.0 std dev → Stop + "CoachHint_Health_Critical"
- Konsekutive warnings: 3+ warnings på rad → eskalerer til Critical
- Critical strain → automatisk pause øvelse + vis hvile-instruksjon (30 sek minimum)

### 3. Smart Coach adaptiv logikk

Bygg på eksisterende SmartCoachEngine.cs og SmartCoachExerciseAdapter.cs:

**Baseline-beregning (utvid eksisterende):**
- Hent 30-dagers historikk fra DatabaseService (minimum 3 økter, ideelt 14+ dager)
- Beregn baseline for pitch, F1, F2, intonation range, resonance score
- Confidence level: "high" (14+ dager), "medium" (7–13 dager), "low" (3–6 dager)
- Auto-oppdater hver 7. dag

**Adaptive målsoner (nøkkellogikk):**
1. Start konservativt fra baseline
2. Øk pitch-target med maksimalt 5 Hz per økt hvis bruker scorer ≥70% i 3 påfølgende økter
3. Default progresjon: 2 Hz/uke. Juster dynamisk (VoiceProfileExtensions.cs-logikk): 3 Hz/uke hvis pitch forbedret >10 poeng, 1 Hz/uke hvis stagnering
4. Hvis resonans <70% baseline: prioriter resonansøvelser, IKKE øk pitch-target
5. Ved strain-deteksjon: senk pitch-target midlertidig med 10 Hz (TemporarilyLowerPitchTarget-mekanisme)

**Øvelsesprioritering (SmartCoachExerciseAdapter pattern):**
- Identifiser svakeste parameter fra siste 10 økter (lavest gjennomsnittsscore)
- Velg øvelser fra VoiceFeminizationExerciseService som matcher svakeste kategori
- Ved health stop: anbefal breathing-øvelser (for jitter/shimmer) eller resonance-øvelser (for pitch press)
- Respekter øvelsesfrekvens: daily > 3x/week (søn/tir/tors) > 2x/week > weekly

**Coach-meldinger (CoachMessageGenerator pattern):**
- Generer tredelt struktur: Hva (observasjon) / Hvorfor (klinisk forklaring) / Hvordan (konkret øvelse)
- Tilpass til user level: begynner får mer detaljer, avansert får kortere, mer tekniske hints
- Inkluder emoji for visuell feedback (✅ success, ⚠️ warning, 🛑 stop)
- Hent alle tekster via LocalizationService.Instance["key"] for norsk/engelsk støtte
- Eksempel begynner pitch-hint: "🎵 Din pitch er 145 Hz – prøv litt høyere mot 165 Hz ved å forestille deg at lyden kommer fra nesebro i stedet for brystet"
- Eksempel avansert resonans-hint: "✨ F2 på 1650 Hz – juster tungeplassering fremover for å nå 1800+ Hz"

### 4. Stemmehelsebeskyttelse

Implementer sanntidsovervåkning med VoiceHealthService-integrasjon:

**Strain-indikatorer (alle må monitores):**
- Jitter >1.5% (konsekutivt i 3 sykluser) → warning
- Shimmer >3.5% (konsekutivt i 3 sykluser) → warning
- Amplitude spike >2.0 standard deviations → immediate warning
- Pitch press: F0 >180 Hz + jitter >1.0% + shimmer >3.0% → immediate stop
- HNR <15 dB → warning (stemme-kvalitetsproblem)

**Reaktiv logikk:**
- 1 warning: vis CoachHint med "ta det rolig"-melding
- 2 konsekutive warnings: senk pitch-target midlertidig (–10 Hz)
- 3 konsekutive warnings ELLER 1 critical indikator → automatisk pause øvelse
- Pause-handling: stopp ExerciseFeedbackEngine, vis modal dialog med hvile-instruksjon, blokker fortsettelse i 15 minutter (lock-out periode)
- Hvile-instruksjon skal inneholde: "🛑 Din stemme trenger hvile. Ta en dyp pust, drikk vann, og slapp av i halsen. Vi fortsetter om 15 minutter."

**Sesjonsovervåkning (VoiceHealthService pattern):**
- Maksimal økttid: 20 minutter (hard limit)
- Anbefalt speaking intervals: 15 min aktiv + 30 sek pause
- Varsle ved 80% terskel (16 min økt, 12 min speaking)
- Spor total daglig treningsbelastning i ProgressionSubsystem

**Health score-beregning:**
- Base: 100 poeng
- Trekk fra: jitter-nivå × 10 + shimmer-nivå × 8 + press-indikatorer × 15
- Kritisk hvis <50; moderat hvis 50–70; OK hvis >70
- Inkluder i FemVoiceScore-beregning (10% vekt)

### 5. Progressiv tilbakemelding og rapportering

**Sanntids feedback-UI (LiveFeedbackView/ViewModel):**
- Fire EvaluationStatus-indikatorer (resonans, pitch, stabilitet, intonasjon) med fargekoding: grønn (Correct), orange (Adjust), rød (Stop)
- Live numeriske verdier: "Pitch: 175 Hz" | "F1/F2: 450/1820 Hz" | "Jitter: 0.8%"
- CoachHint-tekst dynamisk bundet til LocalizationService
- HealthIndicator enum: Safe (grønn), Warning (gul), Critical (rød) med tilhørende ikon
- Oppdateringsfrekvens: maksimalt 20 Hz (50ms throttling) for å unngå UI-flicker

**Øktoppsummering (ExerciseSummaryViewModel pattern):**
- Overall score (0–100%) kalkulert via FemVoiceScore
- Parameter breakdown: Resonance X%, Pitch Y%, Stability Z%, Intonation W%
- Strain level: Low/Moderate/High med beskrivelse ("Lav – Stemmen er avslappet")
- Forbedring vs. forrige økt: "+12% resonance improvement"
- Coach recommendation via SmartCoachExerciseAdapter.GenerateSessionSummaryText()
- Hvis <70% på noen parameter: inkluder spesifikk råd ("Fokus på F2-plassering i neste økt")

**Progresjonsdashboard (ProgressionDashboard.xaml.cs pattern):**
- 4 score bars (Resonance/Pitch/Intonation/VoiceHealth) med visuell bredde (max 200px)
- FemVoiceScore med color-coding: ≥80 grønn, ≥60 gul, ≥40 oransje, <40 rød
- Direction indicators (DirectionAnalyzer): ⬆ increase, ⬇ decrease, ➡ stabilize, ✅ maintain
- 30-day score history som kalenderheatmap (TrendAnalysisService.GenerateCalendarHeatmap())
- Most improved parameter-highlight (sammenlign første vs. siste halv av siste 7 økter)
- Weekly summary: Økter denne uken, total minutter, gjennomsnittsscore, vs. forrige uke

**Baseline-sammenligning (relativ fremgang):**
- Vis alltid forbedring i % relativt til brukerens egen baseline—ikke absolutte normer
- Eksempel: "Resonance: +18% vs. din baseline (1500 Hz → 1770 Hz F2)"
- Hvis baseline confidence "low": vis "Fortsett treningen for mer nøyaktige sammenligninger"

**Smart Coach daglige anbefalinger (SmartCoachViewModel pattern):**
- TodayFocus: resonans/pitch/intonasjon/pust/restitusjon
- TodayRecommendation: lokalisert tekst med konkret øvelse + varighet
- HasHealthWarning flag med tilhørende HealthWarningText
- Active goals (max 3): vis progresjon mot hver (CurrentValue/TargetValue × 100%)
- Weekly progress list: 4-week historikk med pitch/resonance deltas

### 6. Lokalisering og brukeropplevelse

**Språkhåndtering:**
- ALL brukervendt tekst via LocalizationService.Instance["key"]
- Nøkkel-navnekonvensjon: "CoachHint_[Category]_[Situation]" (eks: "CoachHint_Pitch_TooLow")
- Støtt norsk (default) og engelsk—utvid ved behov
- Tilpass UI-oppdateringer ved språkbytte (PropertyChanged event på ViewModel)

**Emoji-bruk (følg CoachMessageGenerator pattern):**
- Suksess: ✅ 🎉 ⭐
- Advarsel: ⚠️ 💡 🔔
- Stopp/hvile: 🛑 😌 💧
- Resonans: ✨ 🌟
- Pitch: 🎵 🎶
- Integrer i coach-meldinger for visuell identifikasjon

**Motivasjon og tone:**
- Bruk positive, støttende formuleringer selv ved corrections
- Ramme strain som "stemmen din trenger hvile" ikke "du gjør noe feil"
- Feir progressive milestones (ProgressionService.milestones): 7-dagers streak, første optimal resonance-score, etc.
- Balancer utfordring med oppnåelighet—unngå frustrerende målsettinger

**Modulstruktur (navngivningskonvensjon fra eksisterende kode):**

└── RealtimeEvaluator.cs             // <100ms evaluering loop, produserer EvaluationResult

└── CoachMessageFormatter.cs         // NYE: Formaterer coach-meldinger med What/Why/How-struktur
```

**Integrasjonspunkter:**

1. **Audio Pipeline:** `AudioCaptureService.OnAudioDataAvailable` → `FormantDetectionService` → `VoiceMetricsCalculator` → `RealtimeEvaluator` → `LiveFeedbackViewModel`

2. **Exercise Session:** `ExerciseDetailViewModel.StartExercise()` → `ExerciseFeedbackEngine.Start()` → kontinuerlig `VoiceMetrics` ingestion → `FeedbackRuleEngine` evaluering → events til UI

3. **Health Monitoring:** `StrainMonitor` subscriber på `VoiceMetrics` stream → detekter warnings → trigger `VoiceHealthService` events → `LiveFeedbackViewModel` reagerer (pause/warning display)

4. **Progression Update:** `ExerciseSummaryViewModel.SaveToDatabase()` → `DatabaseService.CompleteSession()` → `SmartCoachEngine.AssessProgression()` → `AdaptiveTargetZoneService` oppdaterer målsoner

5. **Coaching:** `ProgressionDashboardViewModel.LoadData()` → `SmartCoachEngine.GenerateDailyRecommendation()` → `CoachMessageGenerator` (via DirectionAnalyzer) → lokalisert output til UI
