# Support Package Guide

Support package hjelper utvikler med å feilsøke uten at du må forklare alt manuelt.

## Hva den kan inneholde

- Runtime log.
- Evidence JSON.
- Verification report.
- Audio pipeline diagnostic report.
- Errors-only file.
- Screenshot checklist.
- Session summary.
- App version.
- System summary.
- Settings summary uten secrets.

## Ekskludert som standard

- Profesjonelle fritekstnotater.
- Personlige notater.
- Identifiserbar research export.
- Secrets, tokens eller passord.
- Privat råtekst.

## Hvordan bruke

1. Reproduser feilen hvis mulig.
2. Lag support package fra appen.
3. Legg ved support package sammen med feedback template.
4. Legg ved skjermbilde hvis det er relevant.

## Viktig

Support package er bruker-kontrollert. Den skal ikke lastes opp automatisk.
